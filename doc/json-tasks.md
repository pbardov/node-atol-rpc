# JSON tasks и результаты

## Термины

- **Task** — объект запроса к ДТО АТОЛ (операция: печать чека, открытие смены, отчёт и т.д.).
- **Task result** — структурированный ответ (результат выполнения задачи).

## Где находятся типы

- Типы задач: `src/types/*.task.ts` и связанные файлы.
- Типы результатов: `src/types/*.task-result.ts`.
- Карты соответствий/реестры: `src/types/json-task.map.ts`, `src/types/json-task-result.map.ts`.
- Guard-валидаторы: `src/types/*guard.ts`.

## Валидация структуры

В проекте есть набор утилит для runtime-проверок:

- `structure-validator`
- `describe-validation-errors`
- `get-type-guard-details`

Они используются для того, чтобы:

- раньше обнаруживать несовместимость входных данных с ожидаемым форматом;
- давать более понятные сообщения об ошибках.
