# Архитектура и концепции

## Что такое node-atol-rpc

`node-atol-rpc` — TypeScript-обёртка над RPC-интерфейсом ДТО (драйвера) АТОЛ v10.
Цель библиотеки — дать:

- удобные методы верхнего уровня (`openShift`, `printReceipt`, ...);
- строгие типы для JSON-задач и результатов;
- runtime-проверку структуры входных/выходных данных (type guards/validators).

## Уровни API

- `AtolRpcBase` — низкоуровневый слой работы с JSON-RPC (подготовка запроса, отправка, разбор ответа).
- `AtolRpc` — высокоуровневые методы под конкретные операции.
- `types/*` — типы задач (tasks) и результатов (task results), а также guard-валидаторы.

## JSON task

Под «задачей» понимается JSON-объект, который соответствует формату ДТО АТОЛ (часто его называют «JSON task»).
Библиотека:

1. формирует JSON task согласно типам;
2. (опционально) проверяет структуру перед отправкой;
3. отправляет через RPC;
4. получает и валидирует результат.

См. также: [JSON tasks и результаты](./json-tasks.md).
